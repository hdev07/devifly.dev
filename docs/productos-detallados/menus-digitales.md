# Menús Digitales

Documento interno de venta y claridad comercial.

Objetivo: explicar qué vende este producto, qué resuelve en operación y ventas, qué significa cada función y qué alcance tiene cada plan para no generar expectativas equivocadas.

## 1. Qué es este producto

Menús Digitales es la solución de Devifly para restaurantes, cafeterías, bares y negocios de alimentos que necesitan una carta digital actualizable, clara y fácil de consultar desde celular o QR.

No es solo reemplazar el papel. Es una herramienta para mejorar experiencia, actualizar más rápido y presentar mejor productos y promociones.

## 2. Qué significa la categoría: Restaurantes

La categoría Restaurantes se refiere a soluciones digitales diseñadas para negocios de alimentos con operación en mesa, mostrador o servicio híbrido. El foco está en consulta rápida, claridad visual y facilidad de actualización.

## 3. Qué problema resuelve

- Menús impresos que se ensucian o se vuelven obsoletos.
- Dificultad para cambiar precios o promociones.
- Mala presentación visual de los platillos.
- Clientes que tardan en encontrar lo que buscan.
- Falta de un formato ágil para operación móvil.

## 4. Cómo ayuda a vender más

- Hace más fácil explorar la carta.
- Permite destacar promociones.
- Mejora percepción visual de los productos.
- Reduce fricción al consultar desde QR.
- Ayuda a empujar productos mejor fotografiados o más rentables.

## 5. Qué cubre la solución base del producto

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| QR para mesas o mostrador | El cliente accede al menú escaneando un código | Reduce dependencia de menús físicos y acelera consulta |
| Categorías, fotos y precios | La carta se organiza visualmente con información comercial clara | Facilita decisión de compra |
| Edición sencilla de platillos | El menú puede actualizarse según el alcance contratado | Evita reimprimir o depender de cambios lentos |
| Experiencia móvil rápida | El diseño prioriza lectura y navegación en celular | Mejora uso real dentro del restaurante |

## 6. Planes

## 6.1 Esencial

### Qué resuelve

Resuelve la necesidad de digitalizar una carta pequeña con una experiencia limpia y funcional, sin entrar todavía a administración más avanzada.

### Qué cubre

- Menú digital base.
- Hasta 30 platillos.
- Organización simple de categorías.
- Acceso por QR.

### Qué no cubre

- Carta muy extensa.
- Subcategorías.
- Búsqueda.
- Panel admin robusto.
- Integración con POS.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Hasta 30 platillos | Límite pensado para un menú compacto | Encaja con restaurantes pequeños o cartas simples |
| 3 categorías | Organización básica por grupos principales | Evita que todo aparezca mezclado |
| Acceso por código QR | Ingreso directo al menú desde mesa o mostrador | Facilita consulta inmediata |
| Diseño responsive premium | El menú se adapta y se ve profesional en cualquier dispositivo | Mejora experiencia del cliente |
| 1 foto por platillo | Cada producto puede tener una imagen representativa | Ayuda a vender mejor con apoyo visual |
| Entrega: 1 semana | Tiempo estimado para un alcance compacto | Permite implementar rápido |

### Cuándo conviene venderlo

- Cuando el restaurante es pequeño.
- Cuando la carta es corta.
- Cuando se necesita digitalización rápida.

## 6.2 Profesional

### Qué resuelve

Resuelve la necesidad de administrar una carta más amplia y más dinámica, con categorías, promociones y edición más práctica.

### Qué cubre

- Todo lo del plan Esencial.
- Menú sin límite práctico de platillos dentro del alcance.
- Mejor organización por categorías y subcategorías.
- Búsqueda y panel de edición.

### Qué no cubre

- Operación multi-sucursal.
- Analítica avanzada entre sedes.
- Integración con POS por default.
- Menús por horario complejos tipo franquicia, salvo que se definan.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Todo lo de Esencial | Hereda la base del plan anterior | Permite evolucionar el menú sin perder la estructura inicial |
| Platillos ilimitados | Se amplía capacidad de carga del menú | Sirve para cartas grandes o más vivas |
| Categorías y subcategorías | Organización más granular | Mejora navegación cuando hay muchas opciones |
| Búsqueda de platillos | Herramienta para localizar productos rápidamente | Reduce fricción para clientes con prisa |
| Hasta 3 fotos por platillo | Mayor profundidad visual por producto | Ayuda a comunicar mejor valor o presentación |
| Panel admin para editar menú | Espacio para actualizar elementos según alcance | Reduce dependencia de terceros para cambios frecuentes |
| Sección de promociones | Área dedicada a ofertas destacadas | Ayuda a empujar ventas estratégicas |
| Entrega: 2 semanas | Tiempo estimado para este nivel de complejidad | Alinea expectativas del cliente |

### Cuándo conviene venderlo

- Cuando la carta ya es amplia.
- Cuando se cambia seguido.
- Cuando se quieren destacar promociones y editar contenido con más frecuencia.

## 6.3 Enterprise

### Qué resuelve

Resuelve la operación de cadenas o franquicias que necesitan más estructura, horarios, métricas e integración con sus procesos.

### Qué cubre

- Todo lo del plan Profesional.
- Multi-sucursal.
- Menú por horario.
- Analíticas de consulta.
- Integración con POS según alcance.
- Dominio personalizado.

### Qué no cubre

- ERP completo de restaurante.
- Sistema integral de pedidos, cocina y repartidores por default.
- BI enterprise fuera del alcance definido.
- Soporte operativo ilimitado si no se contrata aparte.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Todo lo de Profesional | Hereda base funcional y operativa del plan anterior | Permite sumar complejidad sin reconstruir desde cero |
| Multi-sucursal / Franquicia | Posibilidad de trabajar varias sedes o unidades | Ayuda a marcas con operación distribuida |
| Menú por horario | Cambios por desayuno, comida o cena | Evita mostrar productos fuera de horario |
| Analíticas de platillos más vistos | Datos sobre consulta o interés por producto | Ayuda a detectar qué llama más la atención |
| Dominio personalizado | La experiencia vive bajo la marca del restaurante | Refuerza profesionalismo |
| Integración con POS | Conexión con sistema de punto de venta según alcance | Ayuda a alinear menú digital con operación comercial |
| Soporte prioritario | Mejor tiempo de atención dentro del acuerdo | Reduce fricción operativa en negocios activos |
| Entrega: 3-4 semanas | Tiempo estimado para un alcance mayor | Ayuda a planear implementación correctamente |

### Cuándo conviene venderlo

- Cuando hay varias sucursales.
- Cuando el menú cambia por horario.
- Cuando la marca necesita más control y más integración.

## 8. Dominio y publicación

| Plan | Tipo de dominio | Ejemplo | Costo |
| --- | --- | --- | --- |
| Esencial | Subdominio Devifly | `turestaurante.devifly.dev` | Incluido sin costo adicional |
| Profesional | Subdominio Devifly | `turestaurante.devifly.dev` | Incluido sin costo adicional |
| Enterprise | Dominio propio (.com o .mx) | `turestaurante.com` | Primer año incluido en el precio del plan |

### Reglas de dominio

- Los planes Esencial y Profesional se publican bajo un subdominio de devifly.dev. No hay costo de dominio.
- Enterprise incluye el registro del dominio propio (.com o .mx) durante el primer año.
- A partir del año 2, la renovación es responsabilidad del cliente. Se puede gestionar a través del servicio Hosting + Dominio ($199 MXN/mes) o de forma independiente.
- El costo de renovación varía según extensión:
  - `.com`: ~$200–$360 MXN/año
  - `.mx`: ~$400–$800 MXN/año
  - Dominios premium o especiales: hasta $1,000+ MXN/año
- Si el cliente ya tiene dominio propio, se configura sin costo adicional de registro; solo se realiza la conexión técnica.
- Cualquier plan puede agregar dominio propio como upgrade pagando el costo de registro del dominio según extensión.
