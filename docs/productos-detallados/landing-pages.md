# Landing Pages

Documento interno de venta y claridad comercial.

Objetivo: explicar qué vende realmente este producto, qué resuelve, qué incluye cada plan, qué significa cada característica y qué queda fuera del alcance base para no sobreprometer.

## 1. Qué es este producto

Landing Pages es la solución de Devifly para negocios que necesitan presentar una oferta de forma clara y convertir visitas en acciones concretas, como mensajes, formularios, cotizaciones o solicitudes de contacto.

No es una página para "tener presencia" únicamente. Es una pieza comercial pensada para:

- captar atención,
- ordenar la propuesta,
- responder objeciones,
- facilitar el contacto,
- mejorar conversión desde tráfico orgánico o pagado.

## 2. Qué significa la categoría: Sitios web

En este producto, la categoría Sitios web se refiere a activos digitales de presentación y conversión. No implica por default un sistema complejo, un panel multiusuario o un software interno. Significa una presencia web profesional enfocada en mostrar una oferta y llevar al usuario a una acción.

## 3. Qué problema resuelve

- El negocio no explica bien lo que vende.
- Las campañas mandan tráfico, pero la página no convierte.
- Todo se atiende por WhatsApp sin contexto.
- El cliente no entiende rápido beneficios, diferenciadores o siguiente paso.
- La marca se ve poco profesional frente a la competencia.

## 4. Cómo ayuda a vender más

- Hace más fácil entender la oferta.
- Reduce fricción para contactar.
- Mejora la percepción de autoridad.
- Ordena beneficios, objeciones y llamados a la acción.
- Aprovecha mejor cada visita que ya llega por anuncios, redes o referidos.

## 6. Planes

## 6.1 One Page

### Qué resuelve

Resuelve la necesidad de lanzar una oferta clara y funcional sin construir una estructura demasiado grande. Es ideal cuando el negocio necesita salir rápido o validar una campaña específica.

### Qué cubre

- Una estructura compacta enfocada en una sola oferta o línea de servicio.
- Un recorrido simple para que el visitante entienda y actúe.
- Integración de contacto principal.
- Adaptación móvil.

### Qué no cubre

- Múltiples páginas o arquitectura extensa.
- CMS o administración compleja.
- Funciones avanzadas de sistema.
- Integraciones especiales no definidas en la cotización.
- Gestión de campañas publicitarias.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Secciones esenciales de conversión | Estructura base con hero, beneficios, prueba social, objeciones y CTA según el caso | Evita páginas vacías o confusas que no llevan a una acción concreta |
| Formulario y CTA principal | Botones y formulario orientados a una sola acción comercial | Evita distracciones y facilita que el usuario deje sus datos o escriba |
| Versión móvil optimizada | Priorización de lectura, botones y distribución para celular | Mejora conversión en el canal donde más usuarios navegan |
| Entrega rápida | El alcance es compacto para reducir tiempos de producción | Permite salir al mercado sin esperar un proyecto más grande |

### Cuándo conviene venderlo

- Cuando el cliente necesita presencia rápida.
- Cuando hay una oferta puntual.
- Cuando el presupuesto es controlado.
- Cuando todavía no se requiere una arquitectura grande.

## 6.2 Multi-Sección

### Qué resuelve

Resuelve escenarios donde una sola página compacta ya no basta para explicar bien el valor del negocio. Sirve cuando el cliente necesita mayor profundidad, más confianza y mejor estructura comercial.

### Qué cubre

- Desarrollo de más bloques y secciones.
- Mejor espacio para argumentos de venta.
- Estructura más sólida para servicios, catálogo básico o validación comercial.
- Preparación más clara para campañas y medición.

### Qué no cubre

- CMS avanzado o portal autoadministrable completo.
- CRM, e-commerce o software interno.
- Flujos de checkout avanzados.
- Automatizaciones complejas.
- Administración mensual continua, salvo contratación aparte.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Múltiples bloques y secciones | Se amplía el recorrido con más contenido, narrativa y jerarquía visual | Permite explicar mejor propuestas con más objeciones o beneficios |
| Estructura para objeciones y confianza | Se agregan áreas para testimonios, diferenciadores, procesos, garantías o casos de uso | Aumenta credibilidad y ayuda a cerrar usuarios indecisos |
| Soporte para catálogo básico o servicios | La landing puede mostrar una oferta más amplia sin llegar a ser una tienda o portal completo | Ayuda a negocios con varias líneas de servicio o productos base |
| Medición para campañas | Se deja lista para captación y análisis básico de resultados según herramientas definidas | Permite saber qué tan bien convierte el tráfico que llega |

### Cuándo conviene venderlo

- Cuando el cliente ya invierte en tráfico.
- Cuando la oferta requiere explicación.
- Cuando se necesita una página más persuasiva.
- Cuando una One Page se quedaría corta.

## 6.3 Portal Avanzado

### Qué resuelve

Resuelve la necesidad de tener una presencia web más completa, con mejor estructura de contenido, posibilidad de gestión y margen real de crecimiento.

### Qué cubre

- Arquitectura más amplia de contenidos.
- Preparación para crecimiento de marca o servicios.
- Posibilidad de CMS o gestión autoadministrable según alcance.
- Trabajo SEO más amplio a nivel técnico.

### Qué no cubre

- Desarrollo de un SaaS, CRM o sistema interno completo.
- Automatizaciones avanzadas por default.
- Integraciones enterprise no definidas.
- Soporte evolutivo continuo fuera de un plan de mantenimiento.
- Marketing de contenidos o SEO continuo mensual.

### Detalle de características

| Característica | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Arquitectura de contenido extendida | Se define una estructura mayor para presentar empresa, servicios, recursos o páginas clave | Evita que el sitio se vuelva limitado demasiado pronto |
| CMS o gestión autoadministrable | El cliente puede actualizar ciertas secciones o contenidos según el alcance pactado | Reduce dependencia para cambios simples de contenido |
| SEO técnico ampliado | Se trabaja una base más sólida de estructura, rastreo, indexación y organización | Mejora preparación para búsquedas y crecimiento orgánico |
| Preparado para crecer | Se diseña con lógica de expansión y no solo como una pieza puntual | Permite evolucionar el sitio sin rehacerlo desde cero tan rápido |

### Cuándo conviene venderlo

- Cuando el negocio quiere una base web más seria.
- Cuando ya hay varias líneas de contenido o servicio.
- Cuando el cliente planea crecer en posicionamiento y estructura.
- Cuando una landing ya no alcanza para lo que quiere comunicar.

### Reglas de dominio

- Los planes One Page y Multi-Sección se publican en un subdominio de devifly.dev. No hay costo de dominio.
- Portal Avanzado incluye el registro del dominio propio (.com o .mx) durante el primer año.
- A partir del año 2, la renovación del dominio es responsabilidad del cliente. Se puede gestionar a través del servicio de Hosting + Dominio ($199 MXN/mes) o por cuenta propia.
- El costo de renovación varía según extensión:
  - `.com`: ~$200–$360 MXN/año
  - `.mx`: ~$400–$800 MXN/año
  - Dominios premium o especiales: hasta $1,000+ MXN/año
- Si el cliente ya tiene dominio propio, se configura sin costo adicional de registro; solo se realiza la conexión técnica.
- Cualquier plan puede agregar dominio propio como upgrade pagando el costo de registro del dominio según extensión.
