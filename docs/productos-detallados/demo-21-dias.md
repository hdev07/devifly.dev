# Paquete Demo 21 Días

Documento interno de venta y política comercial.

Objetivo: definir con claridad qué es el demo, cuánto cuesta, qué incluye, qué no incluye, cuándo darlo gratis, cómo transicionar al plan completo y cómo venderlo en reunión.

---

## 1. Qué es el paquete demo

Es una versión funcional y completa del plan básico de un producto, publicada bajo un subdominio de devifly.dev durante 21 días naturales. Sirve para que el cliente vea su proyecto real funcionando antes de comprometerse con el plan completo.

No es una maqueta. No es un PDF. No es un "así se vería". Es su sitio funcionando con su contenido, visible y compartible.

## 2. Productos donde aplica

| Producto | Plan base que se entrega como demo | Precio regular del plan |
| --- | --- | --- |
| Landing Pages | One Page | $4,999 MXN |
| Menús Digitales | Esencial | $3,999 MXN |
| Web para Salones de Eventos | Esencial | $5,999 MXN |

No aplica para: Invitaciones Digitales, Catálogos Digitales, Desarrollo Custom.

### ¿Por qué estos tres y no los demás?

- **Landing Pages**: el cliente suele dudar del valor de "una sola página" — verla funcionando cierra la venta.
- **Menús Digitales**: el restaurantero necesita ver cómo se ve su carta real en celular para convencerse.
- **Salones de Eventos**: el ticket es alto ($5,999+) y el cliente quiere certeza antes de invertir — ver su salón profesionalizado es el mejor argumento.
- **Invitaciones** no aplican porque el ticket es bajo ($499-$1,599) y el ciclo de venta es rápido; no necesita periodo de prueba.
- **Catálogos** no aplican porque requieren carga de productos que consume mucho tiempo para un demo.

## 3. Precio del demo

| Concepto | Detalle |
| --- | --- |
| **Precio del demo** | **$999 MXN** |
| Qué incluye | Versión funcional completa del plan básico durante 21 días |
| Dónde se publica | Subdominio devifly.dev (ej. `tusalondeventos.devifly.dev`) |
| ¿Se descuenta al contratar? | **Sí. Los $999 se descuentan íntegros del anticipo (50%) del plan contratado** |
| ¿Es reembolsable si no contrata? | No. Cubre el trabajo de configuración y publicación |

### Ejemplo práctico

El cliente quiere un sitio para su salón de eventos (plan Esencial, $5,999 MXN):

| Paso | Monto | Acción |
| --- | --- | --- |
| 1. Paga demo | $999 MXN | Se construye y publica el demo bajo devifly.dev |
| 2. Usa 21 días | — | El cliente ve su sitio funcionando, lo comparte, lo prueba |
| 3. Decide contratar | — | Se acuerda el plan Esencial ($5,999) |
| 4. Anticipo (50%) | $3,000 MXN | El anticipo normal sería $3,000 |
| 5. Se descuenta demo | -$999 MXN | El demo ya pagado se resta |
| 6. Paga diferencia anticipo | **$2,001 MXN** | Solo paga lo que falta del anticipo |
| 7. Contra entrega | $2,999 MXN | Segundo pago al entregar la versión final |

**Total pagado: $999 + $2,001 + $2,999 = $5,999 MXN** → El cliente no pagó nada extra por el demo.

### Ejemplo con plan más alto

Si después del demo el cliente decide subir a un plan superior (ej. Salón Pro $9,999):

| Paso | Monto |
| --- | --- |
| Demo pagado | $999 MXN |
| Anticipo 50% del Pro | $5,000 MXN |
| Se descuenta demo | -$999 MXN |
| Paga diferencia anticipo | $4,001 MXN |
| Contra entrega | $4,999 MXN |
| **Total** | **$9,999 MXN** |

El demo siempre se descuenta, sin importar a qué plan suba.

## 4. Demo gratis — cuándo aplica

A criterio del vendedor, el demo puede darse **sin costo ($0)** como estrategia comercial en estos escenarios:

| Escenario | Por qué conviene |
| --- | --- |
| Prospecto de ticket alto que duda | Un salón que podría comprar plan Pro o VIP ($9,999–$16,999) — regalar el demo de $999 es mínimo vs. el cierre |
| Cliente referido o estratégico | Genera buena voluntad y puede cerrar rápido |
| Primer proyecto en un giro nuevo | Sirve como caso de estudio o portafolio |
| El vendedor detecta alta intención pero freno por desconfianza | El demo elimina la barrera de "¿y si no me gusta?" |

### Reglas del demo gratis

- Se publica en subdominio devifly.dev (no dominio propio).
- Se usan los mismos 21 días de vigencia.
- Si el cliente contrata, el descuento de $999 **no aplica** (porque no pagó el demo).
- El vendedor debe justificar internamente por qué se dio gratis.
- No se ofrece como primera opción. Se usa como herramienta de cierre cuando hay fricción.

## 5. Qué incluye el demo

### Se entrega

| Elemento | Detalle |
| --- | --- |
| Sitio funcional | Versión completa del plan básico del producto, publicada y accesible |
| Contenido real del cliente | Si el cliente proporciona fotos, textos, precios, info — se usa su contenido real |
| Contenido de apoyo | Si el cliente no tiene todo listo, se complementa con estructura sugerida y placeholders claros |
| Subdominio devifly.dev | Se publica bajo una URL como `tunegocio.devifly.dev` |
| Responsive y funcional | Se ve bien en celular, tablet y desktop |
| Formulario o WhatsApp | El canal de contacto funciona desde el demo |
| 21 días de acceso | El sitio permanece activo 21 días naturales desde la publicación |

### No se entrega

| Elemento | Por qué no |
| --- | --- |
| Dominio propio | El demo vive bajo devifly.dev; el dominio propio viene con el plan completo |
| Panel admin | No hay acceso a edición autónoma durante el demo |
| Cambios ilimitados durante los 21 días | Se entrega una versión; no es un servicio de diseño iterativo |
| SEO indexado | El demo no se optimiza para indexación en buscadores |
| Garantía post-demo | La garantía de 15 días aplica solo al proyecto final contratado |
| Integraciones avanzadas | Cotizador, calendario, CRM, pagos, etc. no se incluyen en el demo |

## 6. Qué pasa al terminar los 21 días

### Si el cliente contrata

| Acción | Detalle |
| --- | --- |
| Se acuerda el plan | El cliente elige el plan que mejor le funcione (puede ser uno superior al demo) |
| Se descuenta el demo | Los $999 se restan del anticipo |
| Se personaliza y evoluciona | Se agregan secciones, funcionalidades y contenido según el plan contratado |
| Se migra a dominio propio | Si el plan incluye dominio (Premium/Enterprise/VIP) o lo contrata aparte |
| Se aplican condiciones normales | 2 rondas de revisión, garantía 15 días, esquema 50/50 |

### Si el cliente no contrata

| Acción | Detalle |
| --- | --- |
| El demo se desactiva al día 22 | El subdominio deja de funcionar |
| El contenido se conserva 30 días adicionales | Por si cambia de opinión; después se elimina |
| Los $999 no son reembolsables | Cubren el trabajo de configuración y publicación |
| El cliente puede reactivar después | Si vuelve dentro de los 30 días, se puede retomar sin construir de cero |
| Después de 60 días totales | El proyecto se da de baja definitivamente |

## 7. Cómo venderlo en reunión

### Script principal

> "Mira, entiendo que quieras ver antes de decidir. Por eso tenemos un paquete demo: por $999 te armo tu sitio real, con tu contenido, publicado y funcionando durante 21 días. Lo puedes ver, compartir, enseñar a tu equipo. Y si decides contratar, esos $999 se te descuentan del anticipo — no pagas nada extra. Es literalmente ver tu proyecto real antes de comprometerte."

### Cuando el cliente dice "es que primero quiero ver cómo queda"

> "Perfecto, para eso existe el demo. Por $999 te lo dejo funcionando 21 días con tu contenido real. Si te convence, se aplica a tu proyecto. Si no, solo invertiste $999 en verlo. ¿Le entramos?"

### Cuando el cliente dice "déjame pensarlo"

> "¿Sabes qué? En lugar de pensarlo en abstracto, te propongo algo: paga el demo, ve tu proyecto funcionando y decide con información real, no con suposiciones. Los $999 se te descuentan si contratas. ¿Qué pierdes?"

### Cuando el cliente tiene presupuesto limitado

> "Puedes empezar con el demo por $999 y ya cuando tengas el presupuesto completo, lo activamos. Así no pierdes tiempo y cuando estés listo, arrancamos inmediato porque ya tenemos la base."

### Cuando el cliente compara con la competencia

> "¿El otro te ofrece ver tu proyecto funcionando antes de pagar? Yo sí. Por $999 te lo demuestro con hechos, no con promesas. Si te convence, avanzamos. Si no, solo invertiste $999."

### Cuando conviene ofrecer el demo gratis

> "Sabes qué, me interesa mucho trabajar contigo. Te voy a armar el demo sin costo para que veas cómo queda tu [salón/restaurante/negocio]. Le echo ganas, lo publico y lo revisamos juntos en 21 días. Si te convence, definimos el plan. ¿Te parece?"

## 9. Métricas de éxito del demo

Para saber si el demo está funcionando como herramienta comercial:

| Métrica | Qué medir |
| --- | --- |
| Tasa de conversión demo → plan | ¿Qué % de demos se convierten en contrato? Objetivo: >60% |
| Tiempo promedio de decisión | ¿Cuántos días tarda el cliente en decidir después del demo? |
| Plan que contratan post-demo | ¿Suelen subir de plan o quedarse en el básico? |
| Demos gratis vs pagados | ¿Cuántos se regalan y cuántos se cobran? |
| Demos que no se convierten | ¿Por qué no contrataron? ¿Qué faltó? |

## 10. Reglas internas

1. El demo se construye con la misma calidad que el proyecto final. No es "una versión fea para que compre la buena".
2. El contenido del demo puede reutilizarse directamente en el proyecto final — no se tira trabajo.
3. No se promete que el demo incluye todo lo que tiene el plan completo. Se deja claro qué falta.
4. El vendedor debe comunicar la fecha exacta de expiración del demo al cliente.
5. Si el cliente quiere extender el demo más allá de 21 días, se cobra $499 MXN por 15 días adicionales (máximo 1 extensión).
6. El demo no se ofrece como "descuento" ni como "promoción". Se ofrece como **inversión inteligente para decidir con certeza**.
