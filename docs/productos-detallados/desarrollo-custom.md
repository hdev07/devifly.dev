# Desarrollo Custom

Documento interno de venta y claridad comercial.

Objetivo: explicar cuándo corresponde vender desarrollo custom, qué incluye realmente, qué resuelve, cómo se estructura y qué límites deben quedar claros desde la conversación comercial.

## 1. Qué es este producto

Desarrollo Custom, también presentado como Software a Medida, es la línea de Devifly para proyectos que ya no caben en un producto empaquetado. Aquí la solución se diseña alrededor del proceso real del negocio, no al revés.

Es el producto correcto cuando el cliente necesita:

- lógica propia,
- módulos especiales,
- múltiples usuarios o roles,
- integración entre sistemas,
- automatizaciones complejas,
- paneles internos,
- cobros, procesos o reglas operativas específicas,
- una plataforma SaaS o una herramienta interna.

## 2. Qué significa la categoría: A medida / Custom

La categoría A medida significa que el alcance no está cerrado en un paquete estándar. Se define por objetivos, prioridades, módulos, riesgos, integraciones, complejidad y roadmap.

No se vende como una plantilla. Se vende como una solución construida específicamente para el negocio.

## 3. Qué problema resuelve

- El negocio ya rebasó soluciones genéricas.
- Hay procesos manuales costosos.
- Existen múltiples herramientas aisladas.
- Se pierde tiempo entre Excel, chats y operaciones repetitivas.
- Hace falta centralizar datos, usuarios, permisos o flujos.
- El negocio necesita una plataforma propia para operar o vender.

## 4. Cuándo conviene venderlo en lugar de un top seller

Conviene mover al cliente a custom cuando pasa cualquiera de estas señales:

- necesita más de un módulo conectado,
- quiere usuarios con roles distintos,
- requiere administración interna,
- quiere integraciones con otras plataformas,
- necesita automatizar operaciones completas,
- quiere pagos, contratos, reportes o lógica específica,
- quiere escalar como producto digital o SaaS.

## 5. Qué cubre la propuesta base de desarrollo custom

| Elemento | A qué se refiere | Qué resuelve |
| --- | --- | --- |
| Definición de alcance contigo | Fase donde se aterriza qué se construirá, prioridades y límites | Evita empezar un proyecto complejo sin claridad |
| Roadmap de entregables | División del proyecto por fases, módulos o hitos | Permite controlar tiempos, inversión y expectativas |
| Construcción por fases | El desarrollo no se aborda como bloque ciego, sino por entregables definidos | Reduce riesgo y facilita validación continua |
| Escalable según tu operación | La solución se diseña pensando en crecimiento futuro | Evita que el sistema se quede corto demasiado rápido |

## 6. Qué resuelve comercialmente

- Propiedad sobre la herramienta y el flujo.
- Menos dependencia de terceros.
- Mayor eficiencia operativa.
- Mejor trazabilidad de clientes, datos o procesos.
- Más margen para automatizar y crecer.

## 7. Qué cubre

- Discovery o levantamiento de necesidades.
- Diseño funcional o arquitectónico según el caso.
- Roadmap por fases.
- Desarrollo alineado a objetivos reales del negocio.
- Iteraciones con validación.
- Posibilidad de escalar después.

## 8. Qué no cubre por default

- Alcance ilimitado sin definición previa.
- Integraciones no analizadas.
- Soporte evolutivo indefinido sin contrato de mantenimiento.
- Consultoría de negocio ilimitada fuera del alcance del proyecto.
- Carga masiva de contenido o data cleansing si no se cotiza.
- Infraestructura enterprise avanzada no definida desde el inicio.

## 9. Cómo se explica la cotización

La cotización en custom no sale de una tabla fija. Se calcula con base en:

- cantidad de módulos,
- complejidad funcional,
- nivel de administración,
- integraciones requeridas,
- número de roles o usuarios,
- volumen de datos,
- necesidades de seguridad,
- timeline,
- soporte esperado.

## 10. Familias de solución que suelen entrar en custom

## 10.1 Marketing y Growth

Ejemplos:

- captura de leads con CRM,
- funnels de email,
- chatbots con IA,
- dashboards de métricas,
- sistemas de referidos.

Se venden como custom cuando el cliente necesita que esas piezas se conecten con su operación real o con más de una herramienta.

## 10.2 Automatización

Ejemplos:

- WhatsApp avanzado,
- cotizadores automáticos,
- generadores de PDFs,
- integraciones vía API,
- flujos internos con Make, Zapier o código.

Se venden como custom cuando el valor está en ahorrar tiempo, reducir errores y conectar procesos.

## 10.3 Sistemas internos

Ejemplos:

- CRM corporativo,
- ERP,
- HRM,
- inventario con alertas,
- helpdesk y tickets.

Aquí custom significa alinear la herramienta al proceso real del negocio.

## 10.4 E-commerce y ventas

Ejemplos:

- e-commerce con logística,
- suscripciones,
- marketplace,
- checkout custom,
- panel de afiliados.

Se vende como custom cuando la monetización y la operación requieren lógica propia.

## 10.5 IA aplicada

Ejemplos:

- asistentes con documentos de empresa,
- generadores de contenido,
- análisis de bases de datos,
- asistentes operativos,
- lead scoring.

Aquí el valor comercial está en acelerar trabajo, responder mejor o priorizar oportunidades.

## 10.6 SaaS listos para escalar

Ejemplos:

- agendas tipo Calendly,
- CRMs verticales,
- reservas para hospitalidad,
- analytics de negocio,
- plataformas SaaS personalizadas.

Aquí custom ya no es solo un sistema interno; puede convertirse en un producto comercializable.

## 11. Cómo venderlo sin asustar al cliente

Frase útil:

"Custom no significa complicarte el proyecto. Significa no forzar tu negocio dentro de una plantilla que después se queda corta. Primero definimos lo esencial, lo construimos por fases y lo dejamos listo para crecer."

## 12. Cómo responder objeciones comunes

### "Suena muy grande"

Respuesta sugerida:

"No necesariamente. Puede empezar por una fase pequeña y muy concreta. Lo importante es que la base sí responda al proceso real del negocio."

### "No sé exactamente qué necesito"

Respuesta sugerida:

"Por eso existe discovery. Primero ordenamos el problema, después definimos el alcance y solo entonces pasamos a construir."

### "Quiero saber cuánto cuesta ya"

Respuesta sugerida:

"Se puede estimar un rango, pero para cotizar bien necesitamos entender módulos, usuarios, integraciones y nivel de complejidad. En custom, improvisar precio casi siempre lleva a promesas malas."

## 13. Cómo explicarlo en una reunión

"Si el cliente necesita algo que no cabe bien en un paquete, no conviene estirarlo a la fuerza. Ahí entra desarrollo custom: una solución diseñada alrededor de su operación, construida por fases y pensada para crecer."

## 14. Dominio y publicación

En desarrollo custom, el dominio se define como parte de la cotización del proyecto.

| Escenario | Qué aplica |
| --- | --- |
| El cliente ya tiene dominio propio | Se configura sin costo de registro; solo conexión técnica |
| Se necesita dominio nuevo (.com, .mx) | Se cotiza como parte del proyecto o del servicio Hosting + Dominio |
| Subdominio devifly.dev temporal | Se puede usar durante desarrollo o staging sin costo |

### Referencia de costos de dominio

| Extensión | Rango de costo anual |
| --- | --- |
| `.com` | ~$200–$360 MXN |
| `.com.mx` | ~$300–$500 MXN |
| `.mx` | ~$400–$800 MXN |
| Dominios premium o especiales | Hasta $1,000+ MXN |

- La renovación anual del dominio es responsabilidad del cliente a partir del año 2.
- Se puede gestionar a través del servicio de Hosting + Dominio ($199 MXN/mes).

## 15. Condiciones comerciales

| Concepto | Detalle |
| --- | --- |
| Esquema de pago | Por fases según roadmap definido (típicamente: anticipo + hitos + entrega) |
| Métodos de pago | Transferencia bancaria, depósito o pago en línea según disponibilidad |
| Rondas de revisión por fase | Definidas en el alcance de cada hito o sprint |
| Material requerido del cliente | Depende del proyecto: procesos documentados, accesos, contenido, logo, reglas de negocio |
| Garantía post-entrega | Se define en el contrato según complejidad del proyecto |
| Vigencia de cotización | La cotización aplica por 15–30 días según complejidad y puede ajustarse si cambia el alcance |
| Soporte post-entrega | No incluido por default; se recomienda contratar un plan de mantenimiento |
